Groovy 是什麼？

Groovy 是運行在 JVM（Java Virtual Machine）上的程式語言：

Groovy 可以直接使用 Java 的 Class 與套件，語法則比 Java 精簡。它預設支援動態型別，也允許明確指定型別。

常見用途：

  • Jenkins Pipeline 與 Shared Library
  • Gradle Build Script
  • 自動化腳本
  • 測試程式
  • 建立 DSL

明確指定型別

這是在表達變數預期保存的資料種類，使介面與意圖更清楚。

只寫明確型別不代表整個 Groovy 程式已進行完整的編譯期型別檢查。Groovy 預設仍是動態語言；@TypeChecked 與 @CompileStatic 才會提供更嚴格的靜態檢查。

本章重點

  1. Groovy 是運行於 JVM 的語言，能直接使用 Java 生態系。
  2. def call(Map config) 是 Jenkins Shared Library 常見的入口方法。
  3. def 表示不明確指定型別，但值在執行時仍有實際型別。
  4. 明確型別能改善介面可讀性，但 Groovy 預設仍是動態語言。
  5. 不應省略 def 或明確型別宣告。
  6. 單引號不插值，雙引號會處理 $value 或 ${expression}。
  7. 三引號用於多行字串，是否插值仍由單引號或雙引號決定。
